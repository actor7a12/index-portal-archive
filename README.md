# index-portal-archive

## 项目简介

index-portal-archive 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，而是作为一个集中存放和展示各种静态 HTML 资源的空间。这些页面可以包括信息页、导航页、工具页、存档页等，方便用户通过浏览器直接访问和浏览。

## 目录结构

```
index-portal-archive/
├── pages/           # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/          # 页面依赖的静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md        # 本文件
└── LICENSE          # 仓库许可证
```

- **pages/**: 所有 HTML 页面文件均放置在此目录下，每个文件对应一个独立页面。
- **assets/**: 存放页面所需的公共或专用资源文件，按类型分文件夹管理。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立、自包含的静态页面。页面内容可能涵盖信息展示、功能工具、资源索引等，不局限于特定主题。所有页面均经过基本整理，确保在浏览器中可直接打开使用。

- 页面文件名应具有描述性，便于识别内容。
- 页面之间无强制依赖关系，可单独访问。
- 资源文件尽量内联或统一存放在 assets 目录，保持整洁。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 使用浏览器直接打开 `pages/` 目录下的任意 HTML 文件。
3. 若部署到静态服务器，可将整个仓库根目录作为站点根目录，通过 URL 直接访问对应页面。

## 维护说明

- 欢迎提交新页面或改进现有页面，请遵循以下原则：
  - 页面文件放入 `pages/` 目录。
  - 如需添加资源文件，请放入 `assets/` 对应子目录。
  - 保持页面独立，不依赖外部服务或动态后端。
- 提交前请确保页面无恶意内容、无外部跟踪脚本，且符合开源社区规范。
- 本仓库主要面向个人归档和小范围分享，不保证长期在线服务或更新频率。
